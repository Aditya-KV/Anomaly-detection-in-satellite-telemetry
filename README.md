# Anomaly Detection in Satellite Telemetry

This project implements a hybrid anomaly detection pipeline for satellite telemetry data, specifically designed to process NASA SMAP/MSL and ESA OPS-SAT datasets. The pipeline leverages a combination of unsupervised methods and supervised methods (when labels are available) to identify anomalies in spacecraft operational data.

## 🚀 How to Run It

### Prerequisites
Ensure you have Python installed along with the required libraries.

```bash
pip install pandas numpy matplotlib seaborn scikit-learn
```

### Execution
Run the main script:
```bash
python script.py
```

Upon execution, the script will:
1. Load the telemetry datasets from the `data/` directory.
2. Generate synthetic data for benchmarking.
3. Apply the anomaly detection pipeline.
4. Output terminal logs.
5. Generate an interactive, styled HTML report (`report.html`) containing all logs and visualization plots, which will automatically open in your default web browser.

## 🧠 Explanation of Models Used

The pipeline employs an **Ensemble Approach** that combines multiple anomaly detection algorithms to robustly identify novelties and outliers:

1. **Isolation Forest (Unsupervised):** 
   - *Purpose:* Detects anomalies by isolating observations. It builds an ensemble of random trees and anomalies are identified as instances that have short average path lengths.
   - *Usage:* Works well on feature-engineered rolling windows of the signal.

2. **One-Class SVM (Unsupervised):**
   - *Purpose:* Learns a decision boundary representing the "normal" data and identifies points falling outside this boundary as anomalies.
   - *Usage:* Utilizes an RBF kernel on scaled rolling window features to capture non-linear temporal relationships.

3. **Random Forest Classifier (Supervised):**
   - *Purpose:* When labeled data is available, a supervised Random Forest is integrated into the ensemble.
   - *Usage:* Learns complex patterns of anomalies using comprehensive time-series features (rolling mean, std, diff, energy, etc.).

4. **Ensemble & Smart Thresholding:**
   - The scores from the unsupervised and supervised (if applicable) models are combined.
   - A persistence rule is applied: an alert is only triggered if an anomaly score exceeds the threshold for `k` consecutive time steps, reducing false positives.

## 💻 Code Explanation

### 1. Data Ingestion & Preprocessing
- **SMAP/MSL:** Loaded from `.npy` files and aligned with ground-truth labels from CSVs.
- **OPS-SAT:** Raw telemetry CSVs are ingested. The `opssat_usefulness_report` function evaluates data quality (NaN ratios, zero-variance columns) before processing.
- **Synthetic Data:** The `generate_synthetic_benchmark` function creates controlled data with predefined anomalies (spikes, drifts) for baseline validation.

### 2. Anomaly Injection
For datasets lacking comprehensive labels (like raw OPS-SAT data), the `inject_anomalies_opssat` function artificially injects contextual and point anomalies (spikes, drifts, dropouts, noise bursts) based on standard deviations. This creates a labeled test set for evaluation.

### 3. Feature Engineering
- Extracts a rich set of temporal features from the raw 1D or multidimensional signals.
- Computes rolling metrics: mean, std, min, max, median, quantiles, and signal energy over specified window sizes.

### 4. Output Generation
- Instead of using standard blocking `plt.show()` commands, the codebase utilizes an integrated **HTML Logger**. 
- It captures all `print()` outputs and dynamically encodes `matplotlib` plots to Base64, embedding them directly into a single `report.html` dashboard.

## 🔭 Future Scope

1. **Deep Learning Integration:** Implement LSTMs or Transformer-based autoencoders to better capture long-term temporal dependencies.
2. **Real-Time Streaming:** Adapt the batch-processing pipeline to accept real-time streaming telemetry via MQTT or Kafka.
3. **Multivariate Correlation:** Enhance the feature extraction engine to analyze complex cross-correlations across thousands of telemetry channels simultaneously.
4. **Interactive Dashboarding:** Migrate the static HTML report to a fully interactive frontend (e.g., React, Streamlit) allowing users to dynamically tweak detection thresholds and window sizes.
